# Contribution Guidelines

Hello! Thanks for your interest in Airbender Platform.

## Ways to contribute

1. Open issues: if you find a bug, have something you believe needs to be fixed, or have an idea for a feature, please open an issue.
2. Add context to existing issues: provide screenshots, logs, code snippets, and whatever else might help resolve issues.
3. Report security issues: see [our security policy](./SECURITY.md).

## Pull request policy

At this time, we do not accept external pull requests by default.

If you want to contribute code changes, please contact the project developers first and obtain explicit approval before opening a pull request. Pull requests opened without prior approval may be closed.

## Licenses

If you contribute to this project with prior approval, your contributions will be made to the project under both Apache 2.0 and MIT licenses.

## Code of Conduct

Be polite and respectful.
